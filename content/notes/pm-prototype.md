## 流程总览

```
需求 → 用户路径 → 信息架构 → 低保真 → 高保真 → 交互说明 → 交付开发
```

## 关键原则

**1. 先路径，再页面**

很多人上来就画页面，正确的顺序是先把用户流程图画完，再从流程图里提取出需要哪些页面。

**2. 低保真不要做视觉**

低保真阶段只验证结构和逻辑，不要在这个阶段纠结颜色和字体。

**3. 状态设计不能省**

每个交互组件都要设计：默认、hover、loading、error、empty、success 六种状态。开发最依赖这部分。
