# Claude 协作说明

这是一个 **Obsidian vault**，不是代码仓库。所有文件是中文 markdown 笔记。

## 项目是什么

跨境电商自学知识库，遵循 **Karpathy 方法论**（第一性原理 → 组件拆解 → 系统集成 → 逆向工程）。
入口看 [[README]] 和 [[00-学习路线图/00-总览]]。

## 目录结构与归属

```
00-学习路线图/   四阶段路线图（结构稳定，少改）
01-底层原理/     理论篇：贸易、支付、物流、平台经济（4 篇）
02-选品与市场/   理论篇：选品方法论（5 篇）
03-平台运营/     理论篇：Amazon 运营（5 篇）
04-流量与营销/   理论篇：广告与流量（5 篇）
05-供应链与物流/ 理论篇：供应链管理（6 篇）
06-法规与合规/   理论篇：合规与法务（5 篇）
07-财务与数据分析/ 理论篇：财务与指标（5 篇）

08-实战项目/     ⚠️ 用户亲历的项目记录，Claude 不要替写
09-案例逆向工程/ ⚠️ 用户拆的真实案例，Claude 不要替写
10-工具箱/       ⚠️ 用户实际用过的工具，Claude 不要替写
11-日记/         ⚠️ 用户日更，Claude 不要替写
12-资料库/       ⚠️ 用户读过的外部资料，Claude 不要替写
```

**核心规则**：01-07 是"知识库"（Claude 可写）；08-12 是"经历库"（只能用户写）。Claude 帮忙整理 / 归档 / 关联可以，但不要凭空生成用户的"项目记录""读书笔记""日记内容"。

## 笔记写作规范（01-07 章节统一风格）

每篇按这个骨架：

```markdown
# 标题

> 一句话引子（金句 / 反直觉观点 / 痛点提问）

## 核心概念
（分小节，多用表格、流程图、列表）

## 实战应用
（5 条左右可执行的建议）

## 进阶思考
> 一段"反共识"或"高一层视角"的洞察

## 思考题
（3 个开放式问题，引导用户结合自身情况思考）

---
*参考：[[00-XX总览]] · 关联：[[同级笔记]] · [[跨章节笔记]]*
```

**命名**：01-07 章节内的子篇用 `001-`、`002-` 三位数前缀；总览用 `00-XX总览.md`。

**双链**：链接到其他笔记用 `[[文件名]]`（不带扩展名 / 路径）；跨目录链接用 `[[XX-章节/文件名]]`。

**配图**：mermaid 流程图 / ASCII 框图 / 表格优先；不写 emoji 除非用户明确要求。

## 用户偏好

- 中文回答，简洁直接，不寒暄
- 终端命令需要用户执行的，告诉用户运行而非替执行
- 大批量内容生成前先告知范围 + 让用户确认
- 在 vault 里创建 / 修改文件前不需要额外确认（属于明确范围内的工作）
- 不要写 PR 摘要式总结、不要写"以上完成了 X"这类啰嗦收尾

## 常见任务

- **新增章节笔记**：按上述骨架写，更新对应 `00-XX总览.md` 的"知识节点"列表
- **整理读书笔记**：触发 `readingnotes` skill
- **画流程图 / 思维导图**：触发 `mermaid-visualizer` / `excalidraw-diagram` / `obsidian-canvas-creator` skill
- **日记归档**：从 `11-日记/` 里某天的内容提炼出主题笔记，建议放到 01-07 对应章节
- **周 / 月复盘**：扫 `11-日记/` 同期内容，输出总结

## 不要做的事

- 不要在用户没要求时主动重构 / 美化已有笔记
- 不要给 01-07 的每篇笔记加 emoji 装饰
- 不要把"实战项目""案例""日记"的内容凭空生成 —— 这些必须来自用户真实经历
- 不要在 `MEMORY.md` 或 memory 目录乱写 —— 那是 Claude 的记忆系统，不是项目笔记
